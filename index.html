EAGER — AI Reading Coach
Early childhood literacy analysis powered by multi-agent LLMs
University of Notre Dame
Jul – Dec 2025
Overview

EAGER is an AI-powered mobile application that gives early-childhood teachers real-time, data-driven feedback on shared book-reading sessions. It analyses classroom audio, transcribes speech, and automatically codes teacher behaviour across 17 focal discourse categories defined by the SABR (Systematic Assessment of Book Reading) framework.

My Contributions
  • Redesigned the multi-agent AI pipeline architecture to improve scalability — each of 9 specialist LLM agents handles a dedicated subset of the 17 SABR coding categories
  • Built the audio processing module: file upload, mono conversion, segment splitting for Whisper transcription, and speaker-separated diarisation
  • Engineered a child-interaction "coach" agent and a professional storytelling practice agent using the OpenAI Assistants API with few-shot prompt design
  • Developed the evaluation framework: fuzzy sentence matching, per-category F1/precision/recall scores, and confusion matrix visualisation
  • Integrated Flask REST backend with Firebase Storage / Firestore for real-time result delivery to the Flutter mobile app

Code Highlight — Practice Agent (Child Simulator)
code_practice_agent.py 
class SABRPracticeAgent:
    """Generates targeted practice utterances
    for one SABR code, given a book script.
    Enables teachers to rehearse before class."""
    def __init__(self, code_name,
                 instructions, examples):
        self.code_name = code_name
        self.assistant = client.beta.assistants.create(
            name=f"SABR {code_name} Practice Agent",
            instructions=instructions,
            model="gpt-4o", temperature=0.7
        )
        self.examples = examples  # few-shot pairs

    def generate_practice(self, book_script):
        """Thread-based call: inject few-shot
        examples then stream the book script."""
        thread = client.beta.threads.create()
        for ex in self.examples:
            client.beta.threads.messages.create(
                thread_id=thread.id, **ex)
        client.beta.threads.messages.create(
            thread_id=thread.id, role="user",
            content=f"Practise '{self.code_name}':\n"
                    f"{book_script}")
        run = client.beta.threads.runs.create(
            thread_id=thread.id,
            assistant_id=self.assistant.id)
        return self._poll_result(run, thread.id)

Logic Simulator — GF2
A full compiler pipeline for digital circuit definition files
University of Cambridge
May – Jun 2026
Overview

A complete logic circuit simulator built in Python as part of the Cambridge Engineering 2nd-year computing project. Users write circuit definitions in a custom EBNF-specified language; the system parses, validates, and simulates the circuit, displaying waveforms in a wxPython GUI. The codebase follows a clean compiler-style architecture: Names → Scanner → Parser → Network → Simulator.

My Contributions
  • names.py — Implemented the symbol table: bidirectional string↔ID mapping, unique error-code allocation, and full input validation with typed exceptions
  • scanner.py — Built the lexical analyser from scratch: character-by-character tokenisation, keyword recognition, comment stripping (// … //), and precise line/column error reporting with ^ markers
  • gui.py (partial) — Contributed GUI components for the waveform display and simulation controls using wxPython
  • All test files — Authored the full pytest test suite (1 200+ test lines): scanner, names, parse, network, devices, and monitors — covering edge cases, error paths, and integration scenarios

def get_symbol(self):
    """Tokenise next sequence of characters.
    Returns Symbol with .type and .id set."""
    sym = Symbol()
    self.skip_spaces()
    sym.line, sym.column = (
        self.line_number, self.column_number)

    if self.current_char == "/":
        self.advance()
        if self.current_char == "/":
            self.skip_comment()
            return self.get_symbol()  # recurse

    elif self.current_char.isalpha():
        name = self.get_name()
        sym.type = (self.KEYWORD
            if name in self.keywords_list
            else self.NAME)
        [sym.id] = self.names.lookup([name])

    elif self.current_char == "":
        sym.type = self.EOF

    return sym

def lookup(self, name_string_list):
    """Return IDs, adding unknown names."""
    if not isinstance(name_string_list, list):
        raise TypeError("Must be a list.")
    ids = []
    for name in name_string_list:
        if name in self.names:
            ids.append(self.names.index(name))
        else:
            self.names.append(name)
            ids.append(len(self.names) - 1)
    return ids
